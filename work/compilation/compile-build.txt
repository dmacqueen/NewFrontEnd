Discussion

* It is assumed that compilation ordering required because of dynamic (as opposed to static)
  dependencies, e.g. arrising from state initialization, is a rare issue.
  May need a compilation order "override" feature to deal with these situations.

* existing REPL/CM

   The existing compilation/build model used by CM is ad hoc and not well documented or
   understood, and the compiler bootstrapping system is excessivly complex (and fragile).

   CM was designed to work with the existing system where everything was built within the
   REPL model.  Future CM should not depend on the REPL and its top-level environment.
